模型评估指标

在推荐系统或匹配类任务里，常用的模型评估指标可以分为「预测问题评价指标」「分类问题评价指标」「回归问题评价指标」
在推荐 / 匹配场景里，**“预测、分类、回归”这三类指标并不是按“任务名字”划分的，而是按「模型到底输出什么」**来划分的。

① 预测问题指标 —— 模型输出「一个具体数值」
输出例子

  • 用户 A 明天会在抖音看多少分钟短视频？→ 18.5 分钟
  • 用户 B 对这部电影的评分会是多少？→ 4.3 星
    本质：预测未来数值，所以用时序/预测类指标。

② 分类问题指标 —— 模型输出「是否喜欢 / 点不点」
输出例子

  • 用户 C 会不会点这个商品？→ 会（正例）/ 不会（负例）
  • 这首歌要不要加入喜欢列表？→ 加（正）/ 不加（负）
    本质：二分类（或多分类），只看对还是错。

③ 回归问题指标 —— 模型输出「一个连续值，但当做回归任务来训练」
输出例子

  • 把“停留时长”当回归目标训练，输出 312.7 秒
  • 把“商品成交额”当回归目标训练，输出 ¥257.4
    本质：虽然业务最后可能只取 Top-N，但训练阶段让网络直接拟合数值，所以用回归指标。
    注意：训练完往往再按“得分排序”取 Top-K 推荐，评估排序效果时会再转用分类/ ranking 指标（如 NDCG），但训练阶段仍叫回归任务。

输出数值→回归指标；输出概率→分类指标；预测未来数值→预测指标。

一、预测问题评价指标

均方误差（MSE）Mean Square Error

是预测值与真实值的差的平方的平均值。MSE= 1/n ∑(y_i-y_i尖)^2
其中y_i表示真实值，比如模型的真实，比如模型的label；“y_i尖”是预测值，如常用的y_pred。MSE的范围：[0,+∞)，预测值和真实值完全吻合时，MSE=0，即完美模型（实际模型训练中，基本不会出现的啦！！！）；误差越大，MSE该值越大。

均方根误差（RMSE）Root Mean Square Error

是预测值与真实值的差的平方的平均值的平方根。RMSE=sqrt(1/n ∑(y_i-y_i尖)^2)

平均绝对误差（MAE）Mean Absolute Error

是预测值与真实值的绝对值的平均值。MAE=1/n ∑|y_i-y_i尖|

平均绝对百分比误差（MAPE）Mean Absolute Percentage Error

是预测值与真实值的绝对值的平均值除以真实值的平均值。MAPE=1/n ∑|y_i-y_i尖|/y_i
注意：当真实值y_i中存在数据等于0时，该公式不可用，因分母不能为0

对称平均绝对百分比误差（SMAPE）Symmetric Mean Absolute Percentage Error

是对称平均绝对百分比误差。SMAPE=1/n ∑2|y_i-y_i尖|/(y_i+y_i尖)
注意：当真实值y_i中存在数据等于0时，该公式不可用，因分母不能为0

均方对数误差（MSLE）Mean Squared Log Error

是预测值与真实值的对数差的平方的平均值。MSLE=1/n ∑(log(y_i+1)-log(y_i尖+1))^2

中位绝对误差（MedAE）Median Absolute Error

是预测值与真实值的绝对值的中间值。MedAE=median(|y_i-y_i尖|)

以上7种预测评价指标如何选用？

  1. 单个指标选用：
  • 当看重真实值和预测值间的差的平方时，选用MSE或RMSE
  • 当看重真实值和预测值间的绝对值误差时，选用MAE或MedAE，两者分别是误差的均值和中位数，MAE对极端值比较敏感
  • 当看重真实值的数据中存在量级差，或不同样本的真实值存在量级差，而且更加关注真实值和预测值的百分比差异时，选用MAPE或SMAPE
  • 当y具有随着x进行指数变动的趋势时，选用MSLE
  1. 多个指标搭配使用
  • RMSE与MAE联合使用，可以看出样本误差的离散程度，若RMSE远大于MAE，则可以知道不同样本的误差差别很大

二、分类问题评价指标

2.1单项分类问题指标 混淆矩阵（Confusion Matrix）

✅ 第一层：一张表搞定——混淆矩阵（Confusion Matrix）

真实\预测 正（预测） 负（预测）
正例 TP（真正例）（抓对了） FN（假负例）（漏网之鱼）
负例 FP（假正例）FP（冤枉好人） TN（真负例）（放对好人）
TP=抓对的坏人，TN=放对的好人，FP=冤枉，FN=漏网。
真实\预测 安检员喊“报警” 安检员喊“放行”
真实坏人 TP（抓对了） FN（漏网之鱼）
真实好人 FP（冤枉好人） TN（放对好人）
TP 真正例：真坏被抓住
FN 假负例：真坏被放走
FP 假正例：好人被冤枉
TN 真负例：好人被放行

✅ 第二层：四大基础指标（由混淆矩阵直接算出）

指标 公式 白话含义 什么时候看
准确率 Accuracy (TP+TN) ⁄ (TP+TN+FP+FN) 猜对的占比 类别均衡
精确率 Precision TP ⁄ (TP+FP) “抓的”里多少真坏 怕“误杀”
召回率 Recall/Sensitivity TP ⁄ (TP+FN) 所有真坏里抓多少 怕“漏网”
特异度 Specificity TN ⁄ (TN+FP) 好人里多少被放过 医学筛查常用
  • 准确率（Accuracy）样本不平衡时，准确率就会失效 如：一个总样本中，正样本占90%，负样本占10%，样本是严重不平衡的，此时我们只需要将全部样本预测为正样本就可得到90%的高准确率，但显然这不是我们想要的模型。

✅ 第三层：综合与阈值无关指标

指标 公式/思路 白话含义 优点
F1 分数 2·Precision·Recall ⁄ (P+R) 精确与召回的调和平均 单值综合，均衡P&R
Fβ 分数 带权重版F1，β>1更看召回 可人为调节P-R偏好 自定义
ROC 曲线 以（假阳）FPR为横轴、TPR为纵轴的曲线 不同阈值下“抓坏-冤枉” trade-off 看图一目了然
AUC-ROC ROC曲线下面积 把ROC变成0-1单值 阈值无关，≤0.5≈瞎猜
PR 曲线 以Recall为横轴、Precision为纵轴 类别极不平衡时比ROC更敏感 不平衡首选

四、常用损失函数

损失函数 = 模型犯错的代价函数；不同的任务、不同的“猜错方式”对应不同的损失函数。
交叉熵损失（Cross Entropy）一般针对的是分类问题，而均值平方差损失（MSE）主要针对的是回归问题。
Loss=−[ylogp+(1−y)log(1−p)]

符号 含义 取值
y 真实标签 0 或 1
p 模型预测“正类”概率 0~1

  • 真实 y = 1
    公式只剩 −log p → p 越接近 1，loss 越靠近 0；p 接近 0，loss 飙到 +∞

二、机器学习教程

2.1 机器学习是如何工作的？

机器学习的工作流程
机器学习模型的目标是从数据中找出输入特征与标签之间的关系，基于这些关系做出预测。

1. 数据收集

**收集数据：**这是机器学习项目的第一步，涉及收集相关数据。数据可以来自数据库、文件、网络或实时数据流。

**数据类型：**可以是结构化数据（如表格数据）或非结构化数据（如文本、图像、视频）。

2. 数据预处理

**清洗数据：**处理缺失值、异常值、错误和重复数据。

**特征工程：**选择有助于模型学习的最相关特征，可能包括创建新特征或转换现有特征。

**数据标准化/归一化：**调整数据的尺度，使其在同一范围内，有助于某些算法的性能。

3. 选择模型

**确定问题类型：**根据问题的性质（分类、回归、聚类等）选择合适的机器学习模型。

**选择算法：**基于问题类型和数据特性，选择一个或多个算法进行实验。

4. 训练模型

**划分数据集：**将数据分为训练集、验证集和测试集。

**训练：**使用训练集上的数据来训练模型，调整模型参数以最小化损失函数。

**验证：**使用验证集来调整模型参数，防止过拟合。

5. 评估模型

**性能指标：**使用测试集来评估模型的性能，常用的指标包括准确率、召回率、F1分数等。

**交叉验证：**一种评估模型泛化能力的技术，通过将数据分成多个子集进行训练和验证。

6. 模型优化

**调整超参数：**超参数是学习过程之前设置的参数，如学习率、树的深度等，可以通过网格搜索、随机搜索或贝叶斯优化等方法来调整。

**特征选择：**可能需要重新评估和选择特征，以提高模型性能。

7. 部署模型

**集成到应用：**将训练好的模型集成到实际应用中，如网站、移动应用或软件中。

**监控和维护：**持续监控模型的性能，并根据新数据更新模型。

8. 反馈循环

**持续学习：**机器学习模型可以设计为随着时间的推移自动从新数据中学习，以适应变化。

三、机器学习基础概念

3.1 过拟合与欠拟合

过拟合（Overfitting）： 过拟合是指模型在训练数据上表现非常好，但在测试数据上表现很差。这通常发生在模型复杂度过高、参数过多，导致模型”记住”了训练数据中的噪声或偶然性，而不具备泛化能力。过拟合的模型无法有效应对新数据。
欠拟合（Underfitting）： 欠拟合是指模型在训练数据上和测试数据上都表现不佳，通常是因为模型过于简单，无法捕捉数据中的复杂模式。欠拟合的模型无法从数据中学习到有用的规律。

3.2 训练与测试误差

训练误差（Training Error）： 训练误差是模型在训练数据上的表现，反映了模型是否能够很好地适应训练数据。如果训练误差很大，可能说明模型不够复杂，欠拟合；如果训练误差很小，可能说明模型太复杂，容易过拟合。

测试误差（Test Error）： 测试误差是模型在未见过的数据上的表现，反映了模型的泛化能力。测试误差应当与训练误差相匹配，若测试误差远高于训练误差，通常是过拟合。

3.3 评估指标

准确率 精确率（Precision）和召回率（Recall）
F1 分数： 精确率与召回率的调和平均数，用于综合考虑模型的表现。
